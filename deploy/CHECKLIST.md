# BOM检查系统 - 部署检查清单

## 部署前检查

### 服务器环境

- [ ] 操作系统: Ubuntu 20.04+/CentOS 7+/RHEL 7+
- [ ] CPU: 2核心及以上
- [ ] 内存: 4GB及以上（推荐8GB）
- [ ] 磁盘: 50GB及以上可用空间
- [ ] root或sudo权限
- [ ] 网络连接正常

### 数据库准备

- [ ] MySQL 8.0+ 或 PostgreSQL 12+ 已安装
- [ ] 创建同步数据库: `plm_sync_db`
- [ ] 创建数据库用户并授权
- [ ] 能够连接到PLM生产数据库
- [ ] PLM生产库只读账号已准备

### 网络配置

- [ ] 能够访问PLM生产数据库（IP/端口）
- [ ] 防火墙允许5000端口（API服务）
- [ ] 如使用Nginx，允许80/443端口
- [ ] DNS解析配置（如使用域名）

## 部署过程检查

### 1. 文件上传

- [ ] 项目文件已上传到服务器
- [ ] 文件完整性检查（无缺失）
- [ ] 文件权限正确

### 2. 自动部署脚本

- [ ] 执行 `deploy.sh` 脚本
- [ ] 系统依赖安装成功
- [ ] 应用用户 `bomuser` 创建成功
- [ ] 目录结构创建完成
- [ ] Python虚拟环境创建成功
- [ ] Python依赖包安装成功
- [ ] systemd服务安装成功

### 3. 配置文件

- [ ] 复制 `.env.example` 到 `.env`
- [ ] 配置PLM生产数据库连接
- [ ] 配置PLM同步数据库连接
- [ ] 配置日志级别和路径
- [ ] 检查 `config/database.yaml`
- [ ] 检查 `config/sync_config.yaml`
- [ ] 文件权限设置为600（.env）

### 4. 数据库初始化

- [ ] 执行 `sync/plm_sync.py --init`
- [ ] 同步表结构创建成功
- [ ] 测试数据库连接
- [ ] 检查表是否创建

### 5. 服务启动

- [ ] 启动 `bom-api.service`
- [ ] 启动 `bom-sync.timer`
- [ ] 服务状态正常
- [ ] 无错误日志

## 部署后验证

### API服务验证

- [ ] 访问健康检查端点: `http://localhost:5000/api/health`
- [ ] 返回状态: `{"status": "healthy"}`
- [ ] API日志正常
- [ ] 无错误信息

### 同步功能验证

- [ ] 手动执行同步: `systemctl start bom-sync`
- [ ] 查看同步日志无错误
- [ ] 数据库中有同步数据
- [ ] 同步统计信息正常: `sync/plm_sync.py --stats`

### 定时任务验证

- [ ] 定时器状态: `systemctl status bom-sync.timer`
- [ ] 下次执行时间正确
- [ ] 定时器已启用

### 日志验证

- [ ] API日志文件存在: `/opt/bom_check/logs/api_service.log`
- [ ] 同步日志文件存在: `/opt/bom_check/logs/sync_service.log`
- [ ] 应用日志文件存在: `/opt/bom_check/logs/bom_check.log`
- [ ] 日志目录可写
- [ ] 日志内容正常

### 权限验证

- [ ] 所有文件属于 `bomuser:bomuser`
- [ ] `.env` 文件权限为600
- [ ] 日志目录可写（777）
- [ ] 报告目录可写（777）
- [ ] 临时目录可写（777）
